## 1. 产品概述

### 1.1 产品定位

一款轻量级的心情记录工具，帮助用户记录生活事件与情绪反应，通过心理学知识分析情绪；同时提供"小确幸"清单功能，帮助用户积累愉悦身心的小事，通过完成小事提升心情，追踪记录完成后的情绪变化。

**产品slogan**：记录心情，遇见更好的自己

### 1.2 核心价值

| 核心价值     | 说明                     |
| -------- | ---------------------- |
| **情绪觉察** | 帮助用户识别和命名自己的情绪，提升情绪智力  |
| **情绪分析** | 基于事件类型分析情绪模式，发现情绪规律    |
| **心理调适** | 提供专业的心理学支持和建议，科学应对负面情绪 |
| **个性记忆** | 记住用户偏好和行为模式，提供越来越懂你的服务 |
| **深度洞察** | 透过情绪表象，分析深层心理需求和模式     |

### 1.3 目标用户

| 用户画像  | 特征                       |
| ----- | ------------------------ |
| 职场新人  | 18-30岁，初入职场，面临工作压力和角色转换  |
| 焦虑人群  | 25-40岁，工作/生活压力大，需要情绪管理工具 |
| 心理成长者 | 20-35岁，对心理学感兴趣，希望自我探索和成长 |
| 冥想爱好者 | 20-45岁，练习正念冥想，需要记录心情变化   |

### 1.4 竞品分析

| 竞品     | 优点           | 不足          |
| ------ | ------------ | ----------- |
| Daylio | 快速记录、统计数据可视化 | 无心理学知识、界面简单 |
| 迹迹     | 情绪曲线、社区功能    | 需要账号、数据不在本地 |
| 吾记     | 社交属性强        | 隐私性弱、功能复杂   |

**我们的差异化**：心理学知识 + 小确幸系统 + 本地隐私 + 深层心理分析

#### 3.3.2 心理防御机制识别

| 防御机制 | 识别信号         | 描述           |
| ---- | ------------ | ------------ |
| 压抑   | 情绪记录少、回避特定话题 | 将不愉快情绪压入潜意识  |
| 否认   | 记录事件与情绪不符    | 拒绝承认某些情绪     |
| 投射   | 将情绪归因于外部     | 认为情绪来自他人而非自己 |
| 合理化  | 过度解释事件       | 为情绪找逻辑理由     |
| 升华   | 将负面转为正向      | 通过创作/运动转化情绪  |
| 退行   | 记录中的依赖信号     | 表现出更幼稚的行为模式  |

## 4. 心理学理论框架

### 4.1 马斯洛需求层次理论应用

```
┌─────────────────────────────────────────────────────────┐
│                    马斯洛需求层次                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│                         ┌─────────┐                     │
│                        │ 自我实现 │ 成长、意义、目标    │← 最高需求
│                        └────┬────┘                     │
│                         ┌────┴────┐                     │
│                        │  尊重需求  │ 成就、被认可       │
│                        └────┬────┘                     │
│                         ┌────┴────┐                     │
│                        │  社交需求  │ 归属、爱、友谊    │
│                        └────┬────┘                     │
│                         ┌────┴────┐                     │
│                        │  安全需求  │ 安全、稳定、财务   │
│                        └────┬────┘                     │
│                         ┌────┴────┐                     │
│                        │  生理需求  │ 睡眠、饮食、健康  │
│                        └─────────┘                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**需求层次与情绪信号映射**：

| 需求层次 | 未满足信号        | 已满足信号      |
| ---- | ------------ | ---------- |
| 生理需求 | 疲惫、烦躁、注意力不集中 | 精力充沛、身体舒适  |
| 安全需求 | 焦虑、恐惧、担忧、失眠  | 平静、放松、有安全感 |
| 社交需求 | 孤独、悲伤、被拒绝感   | 温暖、满足、归属感  |
| 尊重需求 | 愤怒、羞耻、自我怀疑   | 自信、自豪、有价值  |
| 自我实现 | 空虚、无聊、迷茫     | 充实、平静、有意义  |

### 4.2 心理防御机制理论

**成熟防御机制**（推荐）：

| 机制 | 表现           | 识别信号         |
| -- | ------------ | ------------ |
| 升华 | 将负面情绪转化为正向行动 | 运动后心情变好、创作表达 |
| 幽默 | 用幽默化解紧张      | 能自嘲、调侃中释放压力  |
| 认同 | 向榜样学习        | "我会像X一样处理"   |
| 压抑 | 有意识控制情绪      | 能延迟反应、冷静处理   |

**中间防御机制**（注意）：

| 机制  | 表现       | 识别信号            |
| --- | -------- | --------------- |
| 压抑  | 否认或忽略情绪  | 情绪记录少、说"没事"     |
| 合理化 | 为情绪找借口   | "因为...所以我才会..." |
| 投射  | 认为情绪来自他人 | "他让我很生气"        |

**不成熟防御机制**（需关注）：

| 机制  | 表现       | 识别信号          |
| --- | -------- | ------------- |
| 否认  | 拒绝承认情绪   | 说"我没生气"但记录愤怒  |
| 退行  | 表现幼稚行为   | 情绪化、依赖、逃避     |
| 躯体化 | 身体症状表达情绪 | 频繁身体不适、无器质性原因 |

### 4.3 情绪时序模式

```
情绪链分析：

正向循环：
「完成小事」 → 「愉快」 → 「平静」 → 「能量恢复」 → 「更好应对挑战」

负向循环：
「压力事件」 → 「焦虑」 → 「回避」 → 「更焦虑」 → 「挫败感」

干预点：
识别循环中的「干预点」，帮助用户打破负向循环、强化正向循环
```

### 7.3 数据聚合算法

```typescript
// 心情曲线数据聚合
function aggregateMoodCurve(
  records: MoodRecord[],
  timeRange: TimeRange,
  granularity: 'hour' | 'day' | 'week'
): MoodCurveData {
  // 1. 过滤时间范围内的记录
  const filtered = records.filter(r =>
    r.createdAt >= timeRange.startDate &&
    r.createdAt <= timeRange.endDate
  );

  // 2. 按粒度聚合
  switch (granularity) {
    case 'hour':
      return aggregateByHour(filtered);
    case 'day':
      return aggregateByDay(filtered);
    case 'week':
      return aggregateByWeek(filtered);
  }
}

// 需求满足度计算
function calculateNeedSatisfaction(
  records: MoodRecord[],
  need: NeedLevel
): NeedSatisfaction {
  // 1. 筛选与该需求相关的记录
  const relatedRecords = filterByNeed(records, need);

  // 2. 分析情绪信号
  const positiveSignals = countPositiveSignals(relatedRecords, need);
  const negativeSignals = countNegativeSignals(relatedRecords, need);

  // 3. 计算满足度
  const score = calculateScore(positiveSignals, negativeSignals);

  // 4. 提取证据
  const evidence = extractEvidence(relatedRecords, need);

  return { score, evidence, triggerEvents: [], trend: 'stable' };
}
```
