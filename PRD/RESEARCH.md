## 1. 市场调研

### 1.1 市场概述

**情绪管理市场现状**：
- 全球心理健康应用市场持续增长，预计2027年达到140亿美元
- 中国心理健康市场规模超过100亿元，年增长率超过20%
- Z世代和千禧一代对心理健康关注度显著提升

**用户需求趋势**：
- 从"有病治病"向"预防保健"转变
- 轻量级、便捷化的情绪记录工具更受青睐
- 本地隐私保护成为重要考量因素

### 1.2 目标市场定位

| 市场细分 | 市场规模 | 增长潜力 | 进入时机 |
|----------|----------|----------|----------|
| 职场人群 | 中大型 | 高 | 最佳 |
| 学生群体 | 大型 | 高 | 最佳 |
| 焦虑人群 | 中型 | 高 | 较佳 |
| 心理爱好者 | 小型 | 中 | 一般 |

### 1.3 市场需求分析

**核心需求**：
1. **快速记录** - 随时随地记录心情，无需复杂操作
2. **情绪分析** - 了解自己的情绪模式和规律
3. **心理学指导** - 科学应对负面情绪
4. **隐私保护** - 数据存储在本地，不上传云端

**次要需求**：
1. 小确幸清单管理
2. 数据可视化
3. 成长追踪
4. 个性化推荐

### 2.2 竞品深度分析

#### Daylio（主要竞品）

**优点**：
- 记录速度快（30秒内完成）
- 数据统计可视化好
- Micro Journal概念创新
- 无需写长文字

**缺点**：
- 无心理学知识融入
- 无小确幸功能
- 界面设计较简单
- 无深层心理分析

**用户评价摘录**：
> "记录很方便，但是没有告诉我为什么心情不好" - App Store用户
> "希望能有更多心理学的建议" - Google Play用户

#### 迹迹

**优点**：
- 情绪曲线直观
- 社区功能活跃
- 话题广场有趣

**缺点**：
- 需要注册账号
- 数据不上本地
- 隐私性弱
- 功能过于社交化

#### 吾记

**优点**：
- 社区氛围好
- 话题丰富
- 内容质量较高

**缺点**：
- 隐私性差（社交属性太强）
- 功能复杂
- 记录门槛高

## 3. 用户调研

### 3.1 用户画像

#### 画像一：职场新人 小李
- **年龄**：24岁
- **职业**：互联网公司产品助理
- **场景**：
  - 刚入职，面对新环境压力大
  - 经常加班到很晚
  - 与领导沟通有压力
- **痛点**：
  - 不知道如何调节情绪
  - 找不到合适的方式倾诉
  - 想记录但觉得写日记太麻烦
- **需求**：
  - 快速记录心情
  - 获取专业建议
  - 了解自己情绪变化规律

#### 画像二：焦虑人群 王姐
- **年龄**：35岁
- **职业**：国企中层管理
- **场景**：
  - 工作家庭双重压力
  - 经常失眠、焦虑
  - 需要长期情绪管理
- **痛点**：
  - 现有App功能太浅
  - 担心隐私泄露
  - 想要系统性的分析
- **需求**：
  - 深层心理分析
  - 成长追踪
  - 数据完全私密

#### 画像三：心理成长者 小张
- **年龄**：28岁
- **职业**：心理咨询师（对心理学感兴趣）
- **场景**：
  - 心理学爱好者
  - 喜欢自我探索
  - 希望理论结合实践
- **痛点**：
  - 现有App心理学知识太少
  - 想要更专业的分析
  - 对理论感兴趣
- **需求**：
  - 马斯洛需求分析
  - 防御机制解读
  - 专业术语解释

### 3.2 用户访谈摘要

**访谈对象A（产品经理，26岁）**：
> "我用过好几个心情记录App，但感觉都很表面。Daylio记录方便但没有心理学内容，吾记又有社交功能让我不太舒服。我希望有一个既专业又私密的工具。"

**访谈对象B（大学生，22岁）**：
> "我经常心情不好的时候不知道怎么办，希望能告诉我一些具体的方法，而不是只是说'保持心情愉快'这种废话。"

**访谈对象C（HR，33岁）**：
> "工作压力大的时候我会用冥想App，但有时候也想记录下来。我觉得'小确幸'这个概念很好，能帮助我建立积极的心态。"

### 3.3 用户调研结论

**核心发现**：
1. **快速记录是刚需** - 用户需要在30秒内完成记录
2. **心理学建议要具体** - 用户讨厌空洞的安慰话语
3. **隐私是重要考量** - 越来越多用户关注数据安全
4. **小确幸功能受欢迎** - 用户对积极心理学应用感兴趣
5. **深层分析是差异化点** - 用户愿意为更专业的分析付费/使用

## 4. 技术调研

### 4.1 前端技术趋势

**框架选择**：
- React 18 + TypeScript 已成为行业标准
- Vite 凭借极速热更新正在取代 CRA
- React Server Components 是未来方向（但不适合当前MVP）

**UI方案**：
- Tailwind CSS 凭借开发效率优势持续增长
- Radix UI、Headless UI 提供无障碍支持
- CSS-in-JS 方案（styled-components、emotion）仍然流行

**状态管理**：
- React Context + useReducer 仍是轻量级首选
- Zustand 凭借简洁API获得更多关注
- React Query 适用于服务端状态管理

### 4.2 可视化技术

**图表库对比**：

| 库 | 优点 | 缺点 | 推荐度 |
|----|------|------|--------|
| Recharts | React原生、类型好 | 定制化有限 | ⭐⭐⭐⭐⭐ |
| Chart.js | 功能丰富 | React集成一般 | ⭐⭐⭐ |
| ECharts | 定制化强 | 体积大 | ⭐⭐⭐ |
| D3.js | 无限可能 | 学习曲线陡 | ⭐⭐⭐ |

**结论**：选择 Recharts 作为主要图表库

### 4.3 AI服务调研

**国内免费AI模型对比**：

| 服务商 | 模型 | 免费额度 | API易用性 | 稳定性 | 推荐度 |
|--------|------|----------|-----------|--------|--------|
| 智谱AI | GLM-4 | 100万tokens/月 | 良好 | 良好 | ⭐⭐⭐⭐⭐ |
| 阿里云 | Qwen-turbo | 100万tokens/月 | 优秀 | 优秀 | ⭐⭐⭐⭐⭐ |
| 百度智能云 | ERNIE-4.0 | 500次/天 | 良好 | 良好 | ⭐⭐⭐⭐ |
| 讯飞星火 | Spark-3.5 | 无限 | 一般 | 良好 | ⭐⭐⭐ |

**推荐方案**：
- 主要：智谱AI（免费额度充足，API易用）
- 备选：阿里云通义千问（稳定性高）
- 降级：本地规则引擎（无网络时）

### 4.4 数据存储方案

**localStorage vs IndexedDB**：

| 方案 | 容量 | 性能 | 易用性 | 适用场景 |
|------|------|------|--------|----------|
| localStorage | 5-10MB | 快 | 简单 | 小量数据、设置 |
| IndexedDB | 无限制 | 中 | 复杂 | 大量历史数据 |

**结论**：
- 心情记录、小确幸等业务数据：localStorage（5MB足够）
- 如需存储大量历史数据：IndexedDB降级方案

## 5. 心理学理论框架

### 5.1 马斯洛需求层次理论

**五层次需求与情绪映射**：

```
                    ┌─────────────┐
                   │  自我实现   │ ← 成长、意义、目标
                    └──────┬──────┘
                    ┌──────┴──────┐
                   │   尊重需求   │ ← 成就、被认可
                    └──────┬──────┘
                    ┌──────┴──────┐
                   │   社交需求   │ ← 归属、爱、友谊
                    └──────┬──────┘
                    ┌──────┴──────┐
                   │   安全需求   │ ← 安全、稳定、财务
                    └──────┬──────┘
                    ┌──────┴──────┐
                   │   生理需求   │ ← 睡眠、饮食、健康
                    └─────────────┘
```

**需求与情绪信号对照**：

| 需求层次 | 未满足信号 | 已满足信号 | 相关标签 |
|----------|------------|------------|----------|
| 生理需求 | 疲惫、烦躁、注意力不集中 | 精力充沛、身体舒适 | health |
| 安全需求 | 焦虑、恐惧、担忧、失眠 | 平静、放松、有安全感 | work, finance |
| 社交需求 | 孤独、悲伤、被拒绝感 | 温暖、满足、归属感 | family, relationship |
| 尊重需求 | 愤怒、羞耻、自我怀疑 | 自信、自豪、有价值 | work, relationship |
| 自我实现 | 空虚、无聊、迷茫 | 充实、平静、有意义 | growth, study |

### 5.2 情绪轮理论（Plutchik）

**八种基本情绪**：

```
              喜悦              悲伤
               ↑                ↑
               │                │
        欣喜若狂 ←──────── 悲痛欲绝
               │                │
        惊喜 ←──────┌───────→ 惊讶
               │      │         │
        得意 ←──┐    │     ┌──→ 害怕
               │    ↓     │     │
             愤怒 ←──────── 焦虑
               ↑                ↑
              厌恶            忧虑
```

**情绪强度等级**：
- 快乐：满意 → 愉快 → 开心 → 欣喜 → 狂喜
- 悲伤：忧郁 → 难过 → 伤心 → 悲痛 → 绝望

### 5.3 心理防御机制

**成熟防御机制**（积极）：

| 机制 | 表现 | 识别信号 |
|------|------|----------|
| 升华 | 将负面情绪转化为正向行动 | 运动后心情变好、创作表达 |
| 幽默 | 用幽默化解紧张 | 能自嘲、调侃中释放压力 |
| 认同 | 向榜样学习 | "我会像X一样处理" |
| 压抑 | 有意识控制情绪 | 能延迟反应、冷静处理 |

**中间防御机制**（注意）：

| 机制 | 表现 | 识别信号 |
|------|------|----------|
| 压抑 | 否认或忽略情绪 | 情绪记录少、说"没事" |
| 合理化 | 为情绪找借口 | "因为...所以我才会..." |
| 投射 | 认为情绪来自他人 | "他让我很生气" |

**不成熟防御机制**（需关注）：

| 机制 | 表现 | 识别信号 |
|------|------|----------|
| 否认 | 拒绝承认情绪 | 说"我没生气"但记录愤怒 |
| 退行 | 表现幼稚行为 | 情绪化、依赖、逃避 |
| 躯体化 | 身体症状表达情绪 | 频繁身体不适 |

### 5.4 认知行为理论（CBT）

**认知、情绪、行为三角**：

```
     认知
    ↙    ↘
   ↓        ↓
情绪 ←────→ 行为
```

**常见认知扭曲**：
1. 非黑即白思维
2. 过度概括
3. 心理过滤
4. 否定正面思考
5. 跳跃式结论
6. 放大或缩小
7. 情绪推理
8. 应该陈述
9. 标签化
10. 个人化
