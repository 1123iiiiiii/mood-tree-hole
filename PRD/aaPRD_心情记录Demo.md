## 1. 产品概述

### 1.1 产品定位

一款轻量级的心情记录工具，帮助用户记录生活事件与情绪反应，通过心理学知识分析情绪；同时提供"小确幸"清单功能，帮助用户积累愉悦身心的小事，通过完成小事提升心情，追踪记录完成后的情绪变化。

**产品slogan**：记录心情，遇见更好的自己

### 1.2 核心价值

| 核心价值 | 说明 |
|----------|------|
| **情绪觉察** | 帮助用户识别和命名自己的情绪，提升情绪智力 |
| **情绪分析** | 基于事件类型分析情绪模式，发现情绪规律 |
| **心理调适** | 提供专业的心理学支持和建议，科学应对负面情绪 |
| **个性记忆** | 记住用户偏好和行为模式，提供越来越懂你的服务 |

### 1.3 目标用户

| 用户画像 | 特征 |
|----------|------|
| 职场新人 | 18-30岁，初入职场，面临工作压力和角色转换 |
| 焦虑人群 | 25-40岁，工作/生活压力大，需要情绪管理工具 |
| 心理成长者 | 20-35岁，对心理学感兴趣，希望自我探索和成长 |
| 冥想爱好者 | 20-45岁，练习正念冥想，需要记录心情变化 |

### 1.4 竞品分析

| 竞品 | 优点 | 不足 |
|------|------|------|
| Daylio | 快速记录、统计数据可视化 | 无心理学知识、界面简单 |
| 迹迹 | 情绪曲线、社区功能 | 需要账号、数据不在本地 |
| 吾记 | 社交属性强 | 隐私性弱、功能复杂 |

**我们的差异化**：心理学知识 + 小确幸系统 + 本地隐私

## 5. 心理学知识库

### 5.1 情绪分类体系

基于Plutchik情感轮理论：

```
                    喜悦
                   ↗   ↘
                信任    厌恶
              ↗          ↘
           恐惧  ─── 愤怒
              ↘          ↙
               悲伤
                  ↘
                 惊讶
```

**维度模型**：
- **效价**：正向情绪（愉快）vs 负向情绪（悲伤、愤怒）
- **唤醒度**：高唤醒（愤怒、喜悦）vs 低唤醒（平静、悲伤）

### 5.2 情绪解读库

| 情绪 | 心理学解释 | 身体反应 |
|------|-----------|----------|
| 愉快 | 正向情绪体验，与多巴胺和血清素分泌相关，大脑奖励系统的激活 | 嘴角上扬、心跳加快、精力充沛 |
| 平静 | 身心放松状态，副交感神经活跃，压力激素水平降低 | 呼吸缓慢、肌肉放松、内心安宁 |
| 焦虑 | 对未来潜在威胁的预警反应，适度的焦虑有助于提前准备 | 心跳加速、呼吸急促、肌肉紧张 |
| 悲伤 | 对失去的自然反应，悲伤是对重要关系/事物断裂的情感标记 | 胸闷、哭泣、疲惫感 |
| 愤怒 | 保护边界和应对不公的情绪，愤怒本身是正当的，需要健康表达 | 血流量增加、肌肉紧绷、攻击冲动 |
| 恐惧 | 对危险的本能反应，触发"战斗或逃跑"机制 | 心跳加速、出冷汗、想逃跑 |
| 惊讶 | 对意外事件的即时反应，帮助快速评估新情况 | 瞳孔放大、短暂停顿、注意力集中 |

### 5.3 调适话语库

| 情绪 | 推荐话语 | 使用场景 |
|------|----------|----------|
| 焦虑 | "此刻的感受只是暂时的，我可以一步一步来" | 焦虑发作时默念 |
| 焦虑 | "未来还没来，我能做的是把握现在" | 过度担忧时 |
| 悲伤 | "悲伤说明我在乎，允许自己脆弱也是一种勇气" | 失落时 |
| 悲伤 | "难过是被允许的，但不会永远这样" | 情绪低谷时 |
| 愤怒 | "我感受到愤怒，这是在提醒我需要保护自己的边界" | 愤怒时 |
| 愤怒 | "我可以选择如何回应，而不是被情绪控制" | 想发脾气时 |
| 恐惧 | "恐惧只是大脑的警报，我可以评估真实的风险" | 害怕时 |
| 恐惧 | "即使害怕，我也可以迈出一步" | 想逃避时 |
| 愉快 | "珍惜这份美好的心情，让它成为今天的力量" | 开心时 |
| 平静 | "此刻的宁静是珍贵的，我值得拥有这份平和" | 平静时 |

### 5.4 身心调节活动库

| 情绪 | 活动类型 | 具体建议 | 时长 |

#### 7.3.2 小确幸智能推荐

```typescript
// 基于心情和历史推荐
function recommendSmallHappiness(
  mood: MoodType,
  memory: UserMemory
): SmallHappy[] {
  // 1. 获取心情相关类别
  const relevantCategories = getCategoriesByMood(mood);

  // 2. 获取用户偏好类别
  const preferredCategories = memory.preferences.favoriteCategories;

  // 3. 合并优先级
  const priorityCategories = mergeCategories(relevantCategories, preferredCategories);

  // 4. 过滤已完成的小事(当天)
  const completedIds = memory.learning.completedSmallHappyIds;
  const notCompletedToday = filterNotCompleted(completedIds);

  // 5. 按类别和心情匹配度排序
  return sortByRelevance(notCompletedToday, priorityCategories, mood);
}
```

#### 7.3.3 推荐去重逻辑

```typescript
// 检查是否应该推荐某小事
function shouldRecommend(
  smallHappy: SmallHappy,
  lastCompletedAt?: string
): boolean {
  if (!lastCompletedAt) return true;

  const daysSinceCompleted = differenceInDays(new Date(), new Date(lastCompletedAt));

  switch (smallHappy.frequency) {
    case 'daily':    return daysSinceCompleted >= 1;
    case 'weekly':   return daysSinceCompleted >= 7;
    case 'once':     return false; // 一次性只推荐一次
    default:         return true;
  }
}
```
